Introduction

Estimating evolutionary timescales with multigene data sets is a common exercise in phylogenetic studies. Multigene data sets can be partitioned by gene, codon position, or both. In this tutorial, we refer to “data subsets” as individual genes or any subunit of the multigene data set. The term “partitions” will refer to a group of data subsets.

Although the data subsets can be concatenated and analysed with a single relaxed-clock model, the patterns of among-lineage rate variation can differ between data subsets even when their tree topologies are identical. For instance, the among-lineage rate variation in mitochondrial genes can differ from that of nuclear genes. Therefore, different relaxed-clock models can be assigned to different data subsets in order to improve estimates of evolutionary timescales.

There are a large number of ways in which multigene data sets can be partitioned. A common approach to compare partitioning schemes is to use Bayes factors or likelihood-based criteria for model fit. In most cases, however, it is infeasable to test all possible partitioning schemes, especially with computationally intensive methods of calculating Bayes factors.

ClockstaR estimates the phylogenetic branch lengths of each data subset. The branch-score distance, known as sBSDmin is calculated for every pair of trees as a measure of the difference in their patterns of among-lineage rate variation. These distances are used to infer the best partitioning strategy using the GAP statistic with the PAM clustering algorithm, as implemented in the package cluster (Maechler et al., 2012) (for details of the sBSDmin metric, see Duchene et al., submitted).

1. Software download and installation

Check that R is installed in your system. If not, please download it here and follow the installation instructions.

A cross-platform version of ClockstaR can be downloaded here.

For all the examples in this tutorial we will be using a simulated data set available at the MEEP web site here.

1.1 Installing ClockstaR in R

First install R in your machine. ClockstaR requires R version 2.14 or higher. Open R and type the following code in the prompt to install the some dependencies:

install.packages("geiger")
install.packages("phangorn")
install.packages("tcltk")
install.packages("cluster")
install.packages("doParallel")
install.packages("foreach")
install.packages("Hmisc")

Install ClockstaR with the following command:

install.packages("path to clockstar", type = "source", repos = NULL, dependencies = T)

Where “path to clockstar” is the path to ClockstaR in your machine. The output should look something like this:

install.packages("~/Desktop/clockstar_0.1.tar.gz", type = "source", repos = NULL, dependencies = T)

The installation may produce some harmless warnings. If the installation is completed without errors, load ClockstaR as follows:

library(clockstar)

1.2 Preparing the data for ClockstaR

Unzip the example data folder “Clockstar_example_files.zip”. This will produce a folder named “clockstar_example_files”, which contains 12 alignments in FASTA format and a tree file in Newick format. The alignments contain 20 sequences generated by simulation. Each alignment represents a data subset, and the tree represents the phylogenetic relationships among the taxa.

The data were simulated with one of three patterns of among-lineage rate variation. Therefore, the expected result for this tutorial is that the appropriate number of relaxed-clock partitions is three (k = 3) (for more details, please see Duchene et al., submitted).

2.1 Running ClockstaR in automatic mode

In automatic mode, ClockstaR uses the Gap statistic to select the appropriate number of relaxed-clock partitions. The output is the relaxed-clock partitioning strategy to be used in subsequent phylogenetic analyses. To run ClockstaR in automatic mode, type the command bellow. Replace “~/Desktop/clockstar_example_data” with the path to the folder with the alignments and the phylogenetic tree (use the complete path, instead of the relative path). ClockstaR will now begin the analysis.

The first part of the output is the optimisation of the branch lengths for each data set on the phylogenetic tree in the clockstar_example_data folder.

The parameters for these functions are as follows:

  • files.directory: This is the path to the folder with the alignments and the phylogenetic tree. In this example, the file is called clockstar_example_data and it is stored on the Desktop.

  • out.file.name: The name of the file for the results from the ClockstaR analysis. The default is “clockstar.output”.

  • model.test: Select T to test the substitution model before optimising the branch lengths. Note that the default is F. For the purpose of this tutorial, we are not testing the substitution model because these data these simulated data are expected to be highly informative with the default models. However, when analysing real data the substution model should be chosen carefully because of its impact in the estimation of branch lengths.

  • mode.run: Specifies whether ClockstaR should run in automatic (a) or manual (m) mode.

Alternatively, ClockstaR can be run interactively. Type “run.clockstar()” to specify the parameters above in a pop-up window and at the prompt.

2.2 Inspecting the default output from ClockstaR

A folder will be created with the estimated distances and scaling factors, and two pdf files containing the results. It will be named as specified in section 2.1 above. The items are described below:

  • bsd.plots.pdf : This pdf contatins a histogram of the sBSDmin distances, an image matrix representing which data subsets are assigned to each of the partitions with the selected k, and a dendrogram of the distances among data subsets.

  • clockstar.diagnostics.pdf : This pdf has two plots. The first is the Gap statistic with the PAM algorithm. The second is an image matrix with the data subsets in each row and the k values in the columns, with the colours representing the partition assignment.

  • scaling.factors.csv : A matrix with the scaling factors used for tree comparisson, as described by Duchene et al., (submitted).

  • partitions.txt : A text file with the partition assignments. This file lists the partitions and the data subsets assigned to each to each partition.

  • bsd.distances.csv : A matrix with the sBSDmin distances between all pairs of data subsets.

  • bsd.dendrogram.tre : This is a dendrogram obtained with the sBSDmin distances.

  • optimized.trees : These are the trees for each data subset. The topologies are identical, but they differ in their branch lengths which have been optimised using the phangorn package.

  • models.csv : If mode.run was selected as T in run.clockstar, this file will contain a table with the nucleotide substitution model selected for each data subset according to the Bayesian information criterion.

The first item in bsd.plots.pdf is a histogram of the pairwise sBSDmin among the trees estimated for all data subsets.

The next item is a matrix with colours representing the partitions to which the data subsets are assigned. In this case the best number of partitions is three (k=3). The colours correspond to each partition. Here, data subsets that start with letters A, B, and C form separate partitions.

The next item is a dendrogram based on sBSDmin distances. The tips correpond to the data subsets, which are the alignments in the clockstar_example_data folder.

The file clockstar.diagnostics.pdf can be used for a more detailed analysis of the partitioning strategy. The first plot shows the Gap statistic for different values of k (shown in black), the successive differences in the statistic (Delta Gap statistic, shown in red), and a constant Gap statistic of 0 (shown in blue).

ClockstaR uses the method described by Tibshirani et al. (2001) implemented in the package cluster (Maechler et al., 2012). It involves calculating the delta of successive Gap values, which is the difference in Gap values for values of k ranging from 1 to N-1, where N is the number of data subsets. The optimal k is the lowest vaue for which the difference in the statistic is positive, which corresponds to the first increase in k that leads to a decrease in the Gap statistic. ClockstaR performs 500 bootstap replicates to obtain the uncertainty in the gap statistic, with the standard error shown in the plot. This number of replicates has been considered sufficient for this method Tibshirani et al. (2001), but it can be changed by specifying a numeric argument to the parameter n.b in the run.clockstar() function.

The second plot in clockstar.diagnostics is an image matrix, where the colours represent individual partitions for different values of k. In some cases the optimal k may be very large, so any subsequent phylogenetic analyses can be very time consuming. In such cases, it can be useful to inspect this matrix and choose a lower value for k. For example, in this data set the optimal is k=3, but if this resulted in a very computationally intensive analysis one could try using k=2. The partitioning strategy would involve using two relaxed clocks, one for those with letter “A”, and another for data subsets with letters “B” and “C”. Note that the colours can vary depending on the analysis and the size of the data set, but the results represented are the same.

2.3 Running ClockstaR in manual mode

The manual mode can be used to assign data subsets to a user-specified number of partitions. The user must provide a value for k.

Replace “~/Desktop/clockstar_example_data” with the path to the folder with the alignments and the phylogenetic tree. ClockstaR will begin the analysis.

Note that two additional arguments were given to the run.clockstar() function, “para” and “ncore”. See the details bellow:

  • para: use para = T to enable the parallel version, otherwise use para = F. The default is F.

  • ncore: This is the number of cores. In general, a larger number of cores results in shorter computing time. But this depends on the size of the data set and the hardware.

The computation time can be compared for different number of cores as follows:

clockstar.time <- matrix(NA, 6, 5)
rownames(clockstar.time) <- 1:6
colnames(clockstar.time) <- c("user.self", "sys.self", "elapsed", "user.child", 
    "sys.child")

for (i in 1:6) {
    clockstar.time[i, ] <- system.time(run.clockstar(files.directory = "~/Desktop/clockstar_example_data", 
        out.file.name = paste0("clockstarNcore", i), model.test = F, mode.run = "a", 
        para = T, ncore = i))
}
